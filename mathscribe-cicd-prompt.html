Architecture Decision

olmOCR requires a 7B-parameter VLM — not runnable on Vercel serverless. The production architecture routes through a verified external provider (DeepInfra / Parasail / Cirrascale) via the olmOCR --server flag, keeping Vercel as the stateless API gateway.

Google SSO — Allowlist Auth

NextAuth.js v5 with Google provider. Only lohjohn02@gmail.com is permitted on initial deploy. Allowlist is env-var driven so additional emails can be added without redeployment.

auth.tsimport NextAuth from "next-auth"
import Google from "next-auth/providers/google"

// Allowlist: comma-separated in ALLOWED_EMAILS env var
const ALLOWED = (process.env.ALLOWED_EMAILS ?? "lohjohn02@gmail.com")
  .split(",").map(e => e.trim().toLowerCase())

export const { handlers, signIn, signOut, auth } = NextAuth({
  providers: [Google({
    clientId:     process.env.GOOGLE_CLIENT_ID!,
    clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
  })],
  callbacks: {
    async signIn({ user }) {
      const email = user.email?.toLowerCase() ?? ""
      if (!ALLOWED.includes(email)) return "/auth/denied"
      return true
    },
    async session({ session, token }) {
      session.user.id = token.sub!
      // Attach paid-tier flag from token
      session.user.paidTier = token.paidTier as string | undefined
      return session
    },
    async jwt({ token, user }) {
      if (user) token.paidTier = "free"  // upgraded via /api/upgrade
      return token
    },
  },
  pages: {
    signIn:  "/auth/signin",
    error:   "/auth/error",
  },
})

olmOCR Integration — Remote Provider Mode

Since Vercel serverless has no GPU, the Next.js API route acts as a thin proxy: it validates the session, constructs the OpenAI-compatible payload (identical to olmOCR's build_no_anchoring_v4_yaml_prompt()), and forwards to the user's selected provider.

app/api/enhance/route.tsimport { auth }  from "@/auth"
import { NextRequest, NextResponse } from "next/server"

// Mirrors olmOCR build_no_anchoring_v4_yaml_prompt() exactly
const OLMOCR_PROMPT = `Attached is one page of a document that you must process. \
Just return the plain text representation of this document as if you were reading it naturally. \
Convert equations to LateX and tables to HTML.\n\
If there are any figures or charts, label them with the following markdown syntax \
![Alt text describing the contents of the figure](page_startx_starty_width_height.png)\n\
Return your output as markdown, with a front matter section on top specifying values for the \
primary_language, is_rotation_valid, rotation_correction, is_table, and is_diagram parameters.`

export async function POST(req: NextRequest) {
  const session = await auth()
  if (!session?.user) return NextResponse.json({ error: "Unauthorized" }, { status: 401 })

  const form = await req.formData()
  const image = form.get("image") as File
  if (!image) return NextResponse.json({ error: "No image" }, { status: 400 })

  // Provider config stored in encrypted session JWT
  const { server, model, apiKey } = session.user.provider ?? {}
  if (!server || !apiKey) {
    return NextResponse.json({ error: "No provider configured", code: "NO_PROVIDER" }, { status: 503 })
  }

  const b64 = Buffer.from(await image.arrayBuffer()).toString("base64")
  const payload = {
    model,
    messages: [{
      role: "user",
      content: [
        { type: "text",      text: OLMOCR_PROMPT },
        { type: "image_url", image_url: { url: `data:image/png;base64,${b64}` } },
      ],
    }],
    max_tokens: 8000,
    temperature: 0.1,
  }

  const res = await fetch(`${server}/chat/completions`, {
    method: "POST",
    headers: { "Content-Type": "application/json", "Authorization": `Bearer ${apiKey}` },
    body: JSON.stringify(payload),
    signal: AbortSignal.timeout(90_000),
  })

  if (!res.ok) return NextResponse.json({ error: `Provider error ${res.status}` }, { status: 502 })
  const data = await res.json()
  const latex = data.choices[0]?.message?.content ?? ""

  return NextResponse.json({ latex, confidence: 0.90, elapsed_ms: 0, model: "olmOCR-2-7B" })
}

CI/CD Pipeline

GitHub Actions workflow — three jobs: lint+test, Claude Code review, Vercel preview deploy. Review runs only on PRs; preview blocks merge until review passes.

  push:
    branches: [main]

env:

      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4

      - run: npm run test -- --coverage --reporter=json > test-results.json

    permissions:
      contents: read

      - uses: actions/checkout@v4

            --file pr.diff \

Test Generation Prompt

.claude/test-gen-prompt.mdYou are generating Vitest unit tests for a TypeScript/Next.js codebase.

## RULES
1. Generate tests ONLY for functions in the diff — do not invent tests for unchanged code
2. Each test must assert ONE behaviour (AAA pattern: Arrange / Act / Assert)
3. Mock external calls (fetch, olmOCR, Google OAuth) — never make real network calls
4. For each happy-path test, generate one edge-case test (empty input, timeout, 4xx response)
5. Name format: describe("functionName", () => { it("does X when Y", ...) })

## SKIP
- Do not generate snapshot tests
- Do not test internal implementation details (private class members)
- Do not generate tests that require a running database

Output a single TypeScript file. No explanation prose.
File path: __tests__/.test.ts

Few-Shot Examples

Two-to-four targeted examples for each ambiguous pattern. The examples show reasoning for why one action was chosen over the plausible alternative — not just the correct output.

Code Review — Security vs. Local Pattern

❌ False positive — DO NOT flag
Pattern: const url = new URL(body.server, process.env.BASE_URL)
Why skipped: The URL constructor uses BASE_URL as the base — user-supplied body.server is the path component only. SSRF is not possible via path injection here. Flag only when user input is the full URL passed directly to fetch().
✓ True positive — flag as CRITICAL
Pattern: const res = await fetch(body.server + "/v1/models")
Why flagged: User-controlled string concatenated directly into fetch URL — classic SSRF. Attacker can reach internal metadata endpoints. Fix: validate body.server against the PROVIDERS allowlist before use.

Test Generation — Coverage Gap Handling

❌ Wrong — generates test for unchanged code
Diff only modifies ocr_vlm.py::infer(). Model generates a test for MathOCR::load_model() which was not in the diff. Skipped: test-gen is scoped to the diff only.
✓ Correct — happy path + edge case pair

olmOCR Prompt — Ambiguous Rotation

Example — rotation correction chain
First attempt: is_rotation_valid: false, rotation_correction: 90
Action: Retry sequentially (not in parallel) with image rotated 90° clockwise. Parallel retries only fire when queue is empty AND the failure is non-rotation (identical to olmOCR process_page() logic).
After retry: is_rotation_valid: true → return result, increment finished_on_attempt_1 metric.

Escalation & Human-in-the-Loop

The pipeline never auto-merges. HITL triggers on three conditions.

TriggerEscalation actionBlocking?
critical finding in review Fail CI, post comment, assign PR to repo owner via CODEOWNERS Yes — merge blocked
review confidence < 0.6 on a warning Flag with low-confidence badge, request human judgment No — advisory
retry exhausted (10 backoff rounds) Exit 1 in CI job, Slack alert via webhook, do not auto-retry Yes — job fails 
scripts/format-review.js// Formats review JSON → GitHub markdown comment
module.exports = (review) => {
  const verdict = {
    approve:         "✅ **Approved** — no issues found",
    request_changes: "⚠️ **Changes requested**",
    block:           "🚫 **Blocked** — critical issues must be resolved",
  }[review.summary.overall_verdict]

  const findings = review.findings.map(f => {
    const icon = { critical:"🚫", error:"❌", warning:"⚠️", info:"ℹ️" }[f.severity]
    const conf = f.confidence < 0.6 ? " _(low confidence — human review requested)_" : ""
    return `${icon} **[${f.severity.toUpperCase()}]** \`${f.file}:${f.line}\`\n`
         + `> ${f.issue}${conf}\n`
         + `> **Fix:** ${f.suggested_fix}`
  }).join("\n\n")

  return `## 🤖 Claude Code Review\n\n${verdict}\n\n${findings || "_No issues found._"}`
}

Retry-with-Error-Feedback Pattern

// POST to Anthropic API — semantic validation retry
async function reviewWithValidation(diff: string, maxRetries = 2) {
  for (let attempt = 0; attempt < maxRetries; attempt++) {
    const result = await callClaudeReview(diff)
    const errors = validateSemantics(result)

    if (errors.length === 0) return result

    // Append specific errors to next prompt — not generic "try again"
    diff += `\n\n## VALIDATION ERRORS FROM PREVIOUS ATTEMPT\n`
         + errors.map(e => `- ${e}`).join("\n")
         + `\nPlease correct these specific issues and resubmit.`
  }
  throw new Error("Review validation failed after retries — human review required")
}

Self-Evaluation Pattern

A second independent Claude instance reviews the generated test file without seeing the original review context — preventing self-review bias. Runs as a separate GitHub Actions step after test generation.

// .github/workflows/ci.yml — test quality gate
- name: Independent test review
  run: |
    # Pass generated tests + original source, NOT the first Claude's reasoning
    claude --model claude-sonnet-4-6 \
      --system-prompt "Review these tests for: coverage of happy + error paths,
        correct mocking of external services, no tests that call real APIs.
        Output JSON: { verdict: 'pass'|'fail', gaps: string[] }" \
      --file __tests__/generated.test.ts \
      --file src/app/api/enhance/route.ts \
      --output-format json > test-review.json
    node -e "const r=require('./test-review.json'); if(r.verdict==='fail') process.exit(1)"
